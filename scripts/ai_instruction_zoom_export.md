# Выгрузка транскриптов созвонов в Markdown

Когда владелец просит выгрузить транскрипт(ы) созвонов в md / markdown / «файлом»:

- **Несколько конкретных дней** (например «за 7, 8, 14, 15, 21, 22, 28, 29 мая») — один вызов **export_zoom_transcripts_markdown** с параметром `dates` = список дат `["2026-05-07","2026-05-08", …]`. Вернёт ВСЕ встречи этих дней в одном файле.
- **Сплошной период** («с 7 по 29 мая», «за неделю») — **export_zoom_transcripts_markdown** с `date_from` и `date_to`. Для одного дня ставь одну и ту же дату в `date_from` и `date_to`.
- **Одна конкретная встреча**, явно указанная владельцем, — **export_zoom_call_markdown** с `call_id`.

Жёсткие правила:
- **НЕ вызывай `list_zoom_calls` и НЕ листай его постранично, чтобы собрать call_id.** Инструмент экспорта сам находит все встречи по `dates`/`date_from`/`date_to` за один вызов. Сбор id вручную — запрещён.
- Один запрос на выгрузку = один вызов экспортного инструмента = один файл. Не делай по одному вызову на встречу.
- Инструмент возвращает `download_url` (публичная ссылка без логина) — отдавай владельцу именно ссылку, а не полный текст транскрипта (длинный текст обрезается клиентом).
- **Ссылка живёт ~30 минут** (`expires_in_seconds` в ответе), потом перестаёт работать. Поэтому отдавай её владельцу сразу в том же ответе. Если ссылка устарела — просто вызови экспорт заново, получится новая.
- Google Drive-дубликаты исключаются автоматически (`include_google_drive` не указывай, если владелец явно не попросил их тоже).
