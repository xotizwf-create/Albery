# Управление доступом к Telegram-агенту

Этот навык позволяет владельцу прямо из Telegram управлять тем, **кто может писать
агенту** (DM-allowlist). Реальный гейт — env `TELEGRAM_ALLOWED_USERS` в
`/root/.hermes/.env` (по умолчанию всем остальным — отказ). Скрипт правит его +
синхронизирует `telegram.allowed_chats` в `config.yaml`, делает бэкап и
перезапускает gateway.

## Кто может управлять доступом

**Только владелец — Александр Никитенко (Telegram id `1451982360`).** Если управлять
доступом просит кто-то другой (включая Евгения) — вежливо откажи и скажи, что выдать/
отозвать доступ может только владелец. Просмотр (`list`) тоже только владельцу.

## Команды

Запускай в терминале (агент работает под root на 186):

```bash
# Показать, у кого сейчас есть доступ
python3 /root/.hermes/scripts/tg_access.py list

# Выдать доступ (по числовому Telegram id; имя по желанию)
python3 /root/.hermes/scripts/tg_access.py add 123456789 "Имя Фамилия"

# Отозвать доступ
python3 /root/.hermes/scripts/tg_access.py remove 123456789

# Найти id нового человека среди недавних отправителей в логе
python3 /root/.hermes/scripts/tg_access.py whoami
```

После `add`/`remove` скрипт сам перезапускает `hermes-gateway` — отдельных действий
не нужно. Передай владельцу строку результата (`✅ Доступ выдан ...`).

## Как выдать доступ новому человеку по @username

Telegram-бот **не может найти человека по @username, пока тот сам не написал боту.**
Поэтому порядок такой:

1. Попроси нового человека нажать **/start** в чате с ботом (он получит отказ — это норма).
2. Запусти `whoami` — увидишь его числовой id среди недавних отправителей `⛔ нет доступа`.
3. Запусти `add <id> "Имя"`.
4. Скажи человеку написать боту ещё раз — теперь доступ есть.

## Важно

- Владельца (`1451982360`) удалить нельзя — скрипт это запрещает.
- Групповые чаты (отрицательные id: home-channel, командная группа) — это
  инфраструктура доставки, **не трогай** их этим навыком (скрипт их сохраняет).
- Дать доступ к этому Telegram-агенту = дать **полные руки** (терминал + боевой MCP).
  Поэтому выдавай доступ только тем, кому владелец доверяет как себе.
