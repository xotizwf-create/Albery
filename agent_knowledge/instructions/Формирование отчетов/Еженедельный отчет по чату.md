# Еженедельный отчет по чату через MCP

Недельный отчет по чату формируется по дневным отчетам активных дней. Активный промпт `chat_weekly_report` остается основным контрактом структуры и JSON.

## Обязательный порядок

1. Получить `get_report_contract(category_key = "chat_weekly_report")`.
2. Найти чат и период недели.
3. Проверить дни недели на наличие сообщений.
4. Для дней с сообщениями прочитать `get_chat_daily_report`; если отчета нет — сформировать daily-отчет по инструкции ежедневного отчета по чату.
5. Для дней без сообщений ничего не формировать и не сохранять `no_data`.
6. Сформировать недельный отчет по имеющимся daily-отчетам активных дней.
7. В `raw_input.skipped_empty_dates` можно указать даты без сообщений.
8. Сохранить через `save_chat_weekly_report` и проверить через `get_chat_weekly_report`.

## Главное правило пустых дней

День без сообщений не является отсутствующим daily-отчетом. Он не блокирует недельный отчет и не требует записи в `chat_daily_reports`.

## Запрещено

- Формировать или сохранять `no_data` daily-отчет за день без сообщений.
- Доформировывать daily-отчет за пустой день только ради полноты недели.
- Строить недельный отчет по сырой недельной переписке вместо daily-отчетов активных дней.
