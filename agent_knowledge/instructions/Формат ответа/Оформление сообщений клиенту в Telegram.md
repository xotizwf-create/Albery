# Оформление сообщений клиенту в Telegram

Тот же стандарт компании, что и в Битриксе («Формат ответа»), но для мессенджера: сообщение
должно «дышать» и читаться с телефона за пару секунд.

## Главное отличие от Битрикса

Telegram отправляется ОБЫЧНЫМ текстом, без разметки. BB-коды (`[b]`, `[i]`, `[URL=…]`),
Markdown (`**`, `#`, таблицы) и HTML клиент увидит буквально, как мусор в сообщении. Поэтому
выделять жирным нечем — структуру держат переносы строк, пустые строки и списки.

## Как оформлять

- Каждая новая мысль — с новой строки. Между смысловыми блоками — ПУСТАЯ строка.
- Перечисления — строками с «• », по одной короткой мысли в пункте.
- Данные из анкеты, условия, цифры — списком, а не одним слипшимся предложением.
- Разделитель «———» отдельной строкой — когда в сообщении два разных блока: например, сначала
  ответ на вопрос, потом ссылка или следующий шаг.
- Вопрос, на который ждёшь ответа, — последней строкой отдельно, чтобы он не потерялся.
- Эмодзи — не больше одного на сообщение и только по делу.

## Чего делать нельзя

- Полотно текста без единого переноса.
- Оформление ради оформления: если ответ — одна фраза («Да, всё верно, передаю дальше»),
  никаких списков и разделителей не нужно. Живой человек так не пишет.
- Заголовки, нумерация разделов, «Вывод:», «Источник:» — это формат отчёта, а не переписки
  с клиентом.

## Пример

```
Дмитрий, вижу вашу анкету:

• магазин — автомобильные лампы и фары
• оборот — 5–20 млн ₽ в месяц
• категория — автотовары

Всё верно?
```
