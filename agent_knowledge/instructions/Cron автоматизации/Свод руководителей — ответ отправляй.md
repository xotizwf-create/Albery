# Что делать на «отправляй» после свода по руководителям

КОГДА владелец отвечает в Telegram «отправляй» / «отправь» / «да» / «ставь» / «ок» **после еженедельного свода по руководителям от cron `leader-digest`** (где был текст «Свод по руководителям за период … (для Евгения)» по каждому руководителю и финал «Отправляем Евгению?») — сразу выполни и **НЕ задавай уточнений**:

1. Возьми текст свода, который владелец только что одобрил (он уже финальный — с учётом всех правок по ходу диалога). Это и есть `digest_text`.
2. Вызови `albery dispatch_leader_evaluations_digest digest_text=<одобренный текст> confirm=true` — без явных date_from/date_to (инструмент сам возьмёт тот же период: прошлая пятница .. сегодня). Он поставит Евгению Палею ОДНУ задачу «Ознакомиться с оценкой руководителей за период <даты>» с дедлайном завтра 10:00 МСК и описанием = текст свода.
3. После успеха верни владельцу короткий человеческий ответ без технических id:
   ```
   Готово — поставил Евгению задачу «Ознакомиться с оценкой руководителей за период <ДД.ММ–ДД.ММ>», дедлайн <ДД.ММ> 10:00 МСК.
   ```

## Что НЕЛЬЗЯ делать

- НЕ вызывай `create_bitrix_task` вручную — для свода есть `dispatch_leader_evaluations_digest`, он сам ставит правильного получателя (Евгений), заголовок и дедлайн.
- НЕ переспрашивай период, получателя или дедлайн — всё собирается автоматически.
- НЕ показывай свод повторно — владелец уже видел и одобрил его в этой же сессии.
- НЕ меняй текст свода при отправке — отправляй ровно одобренный `digest_text`.

## Другие ответы владельца

- «Не отправляй» / «не надо» / «отмена» / «нет» — ничего не создавай. Ответь: «Понял, свод не отправляю.» и закончи.
- «Правки по <имя>: <текст>» — перепиши блок этого руководителя в своде, покажи обновлённый свод целиком и снова задай вопрос «Отправляем Евгению?». НЕ вызывай `dispatch_leader_evaluations_digest`, пока владелец не ответит «отправляй» на обновлённый свод.

## Признаки контекста (как понять, что речь именно про свод руководителей)

Сразу выше в этой же Telegram-сессии было сообщение от cron с шаблоном:
```
Свод по руководителям за период ДД.ММ–ДД.ММ (для Евгения):
<Имя> — <как вёл созвоны: вывод + динамика + эпизод>
———
...
Отправляем Евгению? Ответь «отправляй» — поставлю Евгению задачу...
```

Если такого сообщения в недавнем контексте не было — это другой сценарий, не вызывай `dispatch_leader_evaluations_digest`.
