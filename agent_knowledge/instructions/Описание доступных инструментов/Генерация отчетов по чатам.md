# Генерация отчетов по чатам через MCP

Цель: формировать отчеты быстро и только там, где есть фактическая активность. MCP-агент сам анализирует данные в текущем контексте и не вызывает backend/API endpoint, который сам отправляет данные в LLM.

## Главное правило пустого чата

Если в чате за дату нет сообщений, чат игнорируется:
- не создавать ежедневный отчет;
- не сохранять `no_data`;
- не проверять OCR;
- не искать Zoom;
- не читать предыдущие отчеты ради этого пустого дня;
- не блокировать дневной owner-отчет или недельный отчет отсутствием daily-отчета за пустой день.

Пустой чат — это не источник и не ошибка. Это просто пропуск.

## Быстрый workflow дневного отчета

1. `get_report_contract(category_key = "chat_analysis")` — получить активный контракт отчета.
2. Найти чат через `list_chats`, если `dialog_id` неизвестен.
3. Быстро вызвать `get_chat_transcript(dialog_id, report_date, report_date, include_ocr=false)`.
4. Если сообщений нет — остановиться по этому чату и ничего не сохранять.
5. Если сообщения есть — проверить OCR через `get_chat_ocr_status` и запускать `process_chat_ocr` только если есть вложения без OCR.
6. Получить итоговый transcript через `get_chat_transcript(..., include_ocr=true)`.
7. Читать предыдущий дневной отчет только для continuity; если предыдущий день был без сообщений и отчета нет, это не блокировка.
8. Использовать Zoom-инструменты только когда сообщения/вложения текущего дня дают конкретную связь с Zoom: тема, участник, решение, задача, блокер или риск. Не читать все Zoom по умолчанию.
9. Сформировать отчет по активному `chat_analysis`.
10. Сохранить через `save_chat_daily_report` только если отчет реально сформирован по дню с сообщениями.

## Быстрый workflow недельного отчета

1. `get_report_contract(category_key = "chat_weekly_report")`.
2. Найти чат и период недели.
3. Проверять/формировать daily-отчеты только за дни, где в чате были сообщения.
4. Дни без сообщений не требуют `no_data` и не блокируют недельный отчет.
5. Недельный отчет строить по имеющимся daily-отчетам активных дней; список пустых дат можно указать в `raw_input.skipped_empty_dates`.
6. Сохранить через `save_chat_weekly_report`.

## Для owner daily

Перед owner daily проверять daily-отчеты только по активным чатам, где за `report_date` есть сообщения. Активные чаты без сообщений за дату игнорируются и не считаются недостающим источником.

## Запрещено

- Сохранять `save_chat_daily_report(status="no_data")` для пустого чата.
- Доформировывать daily-отчет за день без сообщений только ради weekly/owner continuity.
- Запускать OCR/Zoom/предыдущие отчеты до первичной проверки, что в чате есть сообщения.
- Вызывать много инструментов «на всякий случай», если transcript-check уже показал пустой чат.
