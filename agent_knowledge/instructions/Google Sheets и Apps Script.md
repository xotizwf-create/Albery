Как делать кнопки, меню и скрипт-автоматизации в Google-таблицах. Следуй буквально — порядок выстрадан реальным инцидентом (2 часа по кругу).

1. СНАЧАЛА изучи реальную структуру: get_google_sheet_meta (вкладки) + read_google_sheet_values по конкретным диапазонам (строки заголовков, первые строки данных). ID таблицы бери ТОЛЬКО из ссылки пользователя в этом диалоге — не по памяти и не со скриншота (OCR искажает ID).
2. Кнопка/меню в таблице = ОДИН привязанный (container-bound) скрипт: manage_apps_script(action='create', parent_id=<id таблицы>), затем action='update' с files=[Code]. Меню — простой триггер onOpen; кнопка — ячейка-чекбокс + onEdit (галочка → расчёт → снять галочку). Оба работают у пользователя сами после перезагрузки таблицы; первый запуск попросит разовое разрешение Google — предупреди об этом.
3. ЗАПУСКАТЬ скрипт через API НЕЛЬЗЯ: action='run' для обычного/привязанного скрипта всегда 404 — ограничение Google, НЕ временный сбой. Не повторяй вызов и не отправляй пользователя «выполнить функцию в редакторе» ради первичной настройки.
4. Первичную настройку и расчёт сделай СВОИМ ходом: read_google_sheet_values (изучение) → write_google_sheet_values (данные/формулы) → format_google_sheet (оформление). Код скрипта нужен только для ПОВТОРНЫХ запусков кнопкой — пользователь сразу должен увидеть готовый результат, не дожидаясь скрипта.
5. Код скрипта — самодостаточный: SpreadsheetApp.getActiveSpreadsheet(), БЕЗ openById других файлов и БЕЗ copyTo между разными таблицами (Google запрещает: «Target range and source range must be on the same spreadsheet»). Данные из другого файла сначала перенеси значениями в эту таблицу своим ходом.
6. ОДИН скрипт на таблицу. Прежде чем создавать новый — найди script_id, созданный ранее в этом диалоге (get_bitrix_bot_chat), и обнови его через action='update'.
7. Блочные структуры (месяцы по 3–4 колонки и т.п.) парси по СТРОКАМ-ЗАГОЛОВКАМ (строка меток + строка подзаголовков), а не по фиксированному шагу: блоки бывают разной ширины. Пустая ячейка источника = пустой результат, не ноль.
8. ПРОВЕРКА ОБЯЗАТЕЛЬНА: после записи прочитай результат назад (read_google_sheet_values) и сверь. Пользователь дал числовой пример («по артикулу X должно быть Y») — прогони ИМЕННО его и покажи в ответе сверку «получилось/ожидалось». Скрипт, который ты не запускал, нельзя называть проверенным: проверка = ты сам посчитал то же самое своим ходом, записал и совпал с эталоном.
