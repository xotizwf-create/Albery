# Известные расхождения в «О компании» (реестр для будущей стандартизации)

Здесь фиксируются известные противоречия между документами «О компании». Для каждого пункта явно указан рабочий источник ответа. Эти пункты помечены для будущего фикса/стандартизации: не «чини» их в ответе и не выдавай неоднозначную роль за установленный факт, но и продолжай отвечать по указанному рабочему источнику.

## Роль «HR» / онбординг / административный старт сотрудника — ОТКРЫТО (зафиксировано 2026-06-16)

**Суть расхождения:**
- «Рабочая матрица решений по ключевым процессам» использует роль **«HR»** (например: «Инициирует: руководитель / HR / Наталья», «Согласует: Наталья, HR / юрист, бухгалтер»).
- Другие документы («Карта процессов модели ИУ», «Описание переходной модели ИУ») и оргструктура говорят, что **отдельной штатной роли HR нет**: подбор/рекрутинг — Артур (+ аутсорс рекрутинга); кадровое делопроизводство — бухгалтерский/кадровый аутсорс-контур; административно-инфраструктурный контур (доступы, роли, реестры, контроль) — Наталья. В оргструктуре есть «Отдел рекрутинга / Адаптация персонала» (Административный блок, рук. Денис Кузьменко), но HR-сотрудник не назначен.

**Как отвечать сейчас:**
- Отвечай **по «Рабочей матрице решений»** — это рабочий источник для вопросов про онбординг, доступы и административный старт: Наталья — владелец контура; руководитель функции / HR инициируют; администратор / координатор исполняет; руководитель функции согласует роль и доступы.
- НО «HR» в матрице — это обозначение функции, а не подтверждённая штатная роль. **Не приписывай «HR» конкретного штатного сотрудника** и не выдавай его за реального человека.
- Когда это уместно по контексту, добавь одной строкой пометку: «(в оргструктуре отдельной штатной роли HR нет — функция распределена; пункт на стандартизацию)». Не раздувай ответ.

**Статус:** ожидает решения владельца — кто канонически закрывает роль «HR» и кто владелец онбординга/адаптации. После решения обновить этот пункт (и, при необходимости, сам документ-матрицу).
