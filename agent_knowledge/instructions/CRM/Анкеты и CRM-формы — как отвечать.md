КОГДА ПРОСЯТ АНКЕТУ / CRM-ФОРМУ (чтобы заявки сразу падали сделками в воронку).

Это НАТИВНЫЕ Битрикс24.Формы. Bitrix REST их СОЗДАВАТЬ НЕ УМЕЕТ — метода нет (crm.webform.add даёт 404, проверено на портале; это ограничение платформы, а не прав вебхука). НИКОГДА не обещай «сейчас создам форму» и не делай свою форму вне Битрикса — это невыполнимо/неверно.

Что делает агент:
1) Вызови list_crm_forms — покажет существующие формы с ПУБЛИЧНОЙ ссылкой (public_url). Если нужная форма уже есть — просто дай её ссылку, её отправляют кандидатам, заявки создают сделки в привязанной воронке.
2) Если формы нет — её создаёт САМ пользователь в интерфейсе Битрикс (агент не может через API). Дай ссылку на конструктор (create_forms_url из list_crm_forms) и проведи по шагам ниже.

=== ПОШАГОВО: как создать CRM-форму, складывающую заявки в воронку ===

ШАГ 0 (главное, здесь чаще всего застревают). СНАЧАЛА создайте поля сделки — без них в форму нечего будет добавить.
CRM → Настройки → Настройки CRM → Пользовательские поля → вкладка «Сделка» → «Добавить поле». Создайте по одному все поля анкеты. Тип поля:
— короткий ответ (имя, Telegram, ссылка на магазин) → «Строка»;
— длинный ответ (категории, комментарий) → «Текст», 2–3 строки;
— число (оборот, процент) → «Число»;
— выбор из вариантов (своя/чужая, есть/нет) → «Список» и впишите варианты.
Включайте «Обязательное» там, где ответ нужен всегда. Сохраните каждое поле.

ШАГ 1. Откройте конструктор: CRM → Ещё (Дополнительно) → CRM-формы → «Создать форму», сценарий «Индивидуальная настройка».

ШАГ 2. Сущности в CRM: выберите «Сделка + клиент». ⚠️ НЕ выбирайте «Клиент» — иначе сделка не создастся, заявка уйдёт только в контакты.

ШАГ 3. Привязка к воронке: раздел «Сущности в CRM» → сущность «Сделка» → выберите вашу ВОРОНКУ и стартовую СТАДИЮ (например «Новая заявка»), назначьте ответственного менеджера.

ШАГ 4. Поля формы: слева «Поля» → «Добавить поле» → появится блок «Сделка» с полями из шага 0 → отметьте нужные галочками → «Добавить». От контакта оставьте «Имя» (и телефон/Telegram, если нужен контакт). Все бизнес-ответы должны быть полями СДЕЛКИ — тогда они попадут в воронку.

ШАГ 5. Дубликаты: «Что делать с найденными дубликатами?» → «Использовать активную сделку, не создавая новую» (повторная заявка не создаст вторую сделку).

ШАГ 6. Действие после отправки: впишите сообщение-«спасибо», например «Спасибо! Мы получили заявку и свяжемся после расчёта».

ШАГ 7. Сохраните → включите «Опубликовано» → «Поделиться» → скопируйте ПУБЛИЧНУЮ ссылку. Её шлют кандидатам.

ШАГ 8. Проверьте тестовой отправкой: в воронке должна появиться сделка. Если поля НЕ видны в карточке сделки (данные при этом СОХРАНЕНЫ, просто скрыты) — откройте сделку → «Настроить форму»/«Настроить поля» карточки → добавьте туда эти поля и сохраните. Повторно отправлять форму не нужно.

Частые заминки (сразу подсказывай):
— Телефон предлагает +34 (Испания): в опубликованной форме нажмите на код слева от поля телефона и выберите «+7 Россия». Это зависит от языка/региона браузера, а не от формы.
— Кнопка «Отправить» не нажимается: в редакторе это только предпросмотр. Сначала «Сохранить», затем открыть форму по публичной ссылке — там кнопка активна.
— «Не могу выбрать поля сделки в форме»: значит поля сделки ещё не созданы — вернитесь к ШАГУ 0.

Коротко: агент НЕ создаёт Битрикс-форму (нет API) — даёт ссылку на готовую (list_crm_forms) либо аккуратно проводит пользователя по шагам 0–8.
