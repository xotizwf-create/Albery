# WB-кабинет: источники и полнота данных

Дашборд не смешивает оперативные события с бухгалтерским итогом:

- заказы — оперативный метод `statistics-api /api/v1/supplier/orders`, с явным диапазоном покрытия;
- реализация, возвраты, комиссия, логистика, хранение, приёмка и прочие удержания — детализация финансовых отчётов `finance-api /api/finance/v1/sales-reports/detailed`;
- реклама — фактические списания `advert-api /adv/v1/upd`;
- прибыль публикуется только после полного финансового отчёта и заполнения себестоимости всех проданных баркодов.

Финансовый бэкфилл всегда запрашивает один шестимесячный диапазон и продолжает его по
`rrdId` до ответа `204`. Курсор каждой страницы сохраняется в `wb_sync_state`, поэтому
рестарт не начинает загрузку заново. Для базового токена WB применяет редкую квоту;
персональный или сервисный токен автоматически включает несколько минутных страниц за тик.
